## Decision Log

| Decision | Rationale | Rejected alternative |
|---|---|---|
| Monochrome warm-neutral palette (stone/cream) over blue base | User chose "Monochrome (neutral)" — Apple/Vogue premium aesthetic. Blue→violet kept only as accent on CTAs/links | Keep blue base — would have been "Brand Blue → Violet" option |
| Centered hero (not split) | User chose "Mesh Gradient (centered)" — most premium editorial feel, matches Bridel template | 2-column split — would have been less premium |
| Hybrid Premium+Tech aesthetic | User chose this — best of both worlds for an Indian real estate SaaS | Pure Bridel (too editorial) or Pure INFINER (too generic) |
| No new dependencies | AGENTS.md emphasizes using what's installed; faster, no bundle bloat | Adding shadcn/ui, tailwind-merge already present, embla-carousel — not needed |
| Bento grid for features | User asked for the best Vercel template per section — INFINER bento is the gold standard | 3-column symmetric grid (current) |
| Serif `Instrument Serif` over Playfair Display | More modern, lighter, Apple-style editorial feel — Playfair is kept as fallback | Keep Playfair (was OK but heavier) |
| `.beam-border` animated conic for highlighted pricing tier | Premium "spotlight" treatment without new deps | Static border (less premium) |
| Hero text reveal via `.line-reveal` (CSS) instead of framer-motion | Works without JS, better for performance + SSR, falls back to visible text | framer-motion only |
| Mesh gradient via pure CSS `radial-gradient` (no WebGL shader) | Zero JS, lightweight, plays nice with `prefers-reduced-motion` | WebGL shader gradient (overkill, +200KB) |

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| **Tailwind v4 `@theme inline` token breakage** — new shadow / radius / mesh tokens may not generate utility classes as expected | Medium | Test each token in isolation after Chunk 1; fall back to direct CSS class if Tailwind doesn't pick it up |
| **Brand-color regression** — the previous `--color-brand-500: #2563eb` was used as a semantic color, now shifted to graphite | Medium | Keep `--color-brand-*` mapped to graphite; add new `--color-accent-*` for blue; existing code using `text-brand-500` will compile fine but render graphite — audit components for `text-brand-500` use |
| **Mesh gradient paint cost on low-end devices** | Low | Use `will-change: transform` only on `.mesh-animated`; keep non-animated variants static; honor `prefers-reduced-motion` |
| **Hero layout shift on font load** | Low | Keep `display: swap` on Inter; preload critical fonts in `layout.tsx` (already done) |
| **Mobile menu focus trap regression** | Low | Test mobile menu keyboard navigation after Chunk 4; current implementation is solid |
| **Inline `<style>` in JSX** — some existing components may inject styles that conflict | Low | Already audited — `trusted-by.tsx` uses global classes; will keep that pattern |
| **Time budget** — 15 chunks is a lot; some may need to be merged | Medium | Each chunk is independent; if running long, can parallelize via subagents (background work) |
